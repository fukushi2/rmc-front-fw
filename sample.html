★フロントエンドフレームワーク★

本フレームワークは、WEBアプリ(※)開発時の画面開発を簡易化するためのview構築をサポートします。

このページは本フレームワークで構築されています。

※WEBアプリ開発…フォーム入力やエラー表示、ページングなどを利用したアプリケーションのこと。
サイトごとにまったく違うビジュアルのWEBサイト制作よりもパワーを発揮します。

Grid System

グリッド数を指定するだけで、クロスブラウザで崩れないレイアウトカラムを実現

max960pxを好きな12グリッドで分割できる「960gs」を採用

SCSS support

メンテナンスしやすいSASS記述によるCSS構築

モジュールごとに追加・修正が容易

テーマ切り替えで簡単に基本スタイルを変更

Graceful Degradation

モダンブラウザでHTML5最高のパフォーマンスを発揮

レガシーブラウザでも必要な機能を損なわない

IE(+各バージョン)用CSSやJSを個別に設定可能

規約

制作における規約を守ることで、メンテナンス・パフォーマンスともに高いコードを利用できます

Rレンダリングを加速

Mメンテナンスが容易

Pパフォーマンスの向上

外部ファイル

RCSSは上に、JSは下に
DOM構築中の再レンダリングやDOM構築ブロックが起こらないため描画が早くなります

MjQueryだけ上に
ページ中の$記述に対応

CSS

Pmin化して１ファイルに
min化してもブラウザ間の差異は出ません

R@importを使わない
CSSから外部CSSを読み込むとレンダリング停止＋通信が走るため描画が遅くなります

M個別ページ専用CSSは直接記述
個別ページのみに１度だけ登場するスタイルは共通CSS化する必要はありません

R小さいアイコンはCSS Spriteで
HTTPリクエスト回数を減らし、画像をキャッシュすれば描画が早くなります

Pexpressionは使わない
IE専用に重い処理を走らせるのはやめましょう

JS

Rライブラリはmin化した内容をまとめて１ファイルに
通信回数を減らして描画が早くなります

Mサイト専用コードもmin化してコメント削除
開発用リソースとリリース用リソースを分けることでメンテナンスが向上します

MJQueryのセレクタ専用にクラスを指定しない
id属性またはdata-*属性を利用します。ただしレンダリングのためにaddClass/removeClassは問題ありません

Monclick,onload,onchange等イベントをインライン記述しない
jQueryの各イベントをbindすれば、ロジックとデザインを分離できます

Pdefer、asyncが使えるコードは属性を指定する
非同期リクエストすることで描画が早くなります

画像

RIMGタグは画像サイズを指定する
サイズがわかっていれば、画像自体が読み込まれなくても画像のスペースを確保できるため描画が早くなります

フォルダ構成

もうファイルの場所で迷わない

ライブラリとアプリケーション用独自コードは別フォルダで管理

  • CSS/
    • lib/外部ライブラリ
      • (jquery-ui-xxx.css)
      • (images/)
    • style.cssアプリケーション用CSS
  • img/
    • logo.png
    • icon/アイコン用フォルダ
  • js/
    • plugins.js外部プラグインを１ファイルにまとめる
    • script.js全ページで共通実行するコード
    • mylib/アプリケーション専用コード・カスタムライブラリ
    • lib/外部ライブラリ
  • sass/
    • style.scssstyle.css書き出し用メインファイル
    • ie.scssIE用CSS（必要に応じて利用）
    • lib/外部ライブラリ
    • themes/アプリケーションテーマおよびモジュールを選択
  • robots.txtクローラ用の指定
  • humans.txt人間用の指定（必要に応じて）
  • xxx.html
